-------------------------------------------------------
0x0090
-------------------------------------------------------
[8-bit] Current Scene Mode
Defines the active game mode: Platforming, Driving, Rail Shooting, or UI / Cutscenes. Use in combination with Current Scene Index (0x0092) to determine the exact 'Scene' the game is currently in.

* Value >= 0x0b means the game is not running an active gameplay scene
* Value < 0x0b means the game is currently running an active gameplay scene

0x00 = Gameplay: Platformer
0x01 = Gameplay: Platformer
0x02 = Gameplay: Platformer
0x03 = Gameplay: Platformer
0x04 = Gameplay: Rail Shooter
0x05 = Gameplay: Platformer
0x07 = Gameplay: Platformer
0x09 = Gameplay: Platformer
0x0a = Gameplay: Car Driving
0x0b = Cutscene or UI Screen
0x0c = Stage Select UI Screen

-------

-------------------------------------------------------
0x0091
-------------------------------------------------------
[8-bit] Current Scene State
0x00 = Start / Reset
0x01 = Active

NOTE: 
 * When this value is 0x00 the game initialises the Scene (see 0x0090 and 0x0092) and then switches the value to 0x01
 * Useful for detecting the exact frame a scene loads or forcing a scene load during debugging

-------------------------------------------------------
0x0095
-------------------------------------------------------
[8-bit] Next Scene Mode
Just like 0x0090 this holds a game mode: Platforming, Driving, Rail Shooting, or UI / Cutscenes. It needs to be used in combination with Next Scene Index (0x0097).

Together with 0x0097, it is used to determine what scene will run after a UI / Cutscene is completed.

-------------------------------------------------------
0x0097
-------------------------------------------------------
[8-bit] Next Scene Index
Just like 0x0092 this identifies a specific Stage Segment, Cutscene or UI Screen.

Together with 0x0095, it is used to determine what scene will run after a Cutscene / UI Screen is completed.
